## Overview
This project is a college information chatbot built with Node.js and Express.js. It uses TensorFlow.js and the Universal Sentence Encoder for semantic intent detection, combined with rule-based shortcuts for common queries. The chatbot handles queries in English and Hinglish (Hindi-English mix), providing responses on topics like admissions, fees, courses, hostel, facilities, placements, library timings, and college schedules.

The application includes a web frontend with multiple pages (Home, About, Courses, Contact) and a chat interface for interacting with the bot. It features conversation memory, spell correction, and structured data responses for better user experience.

---

## Features
- **Natural Language Processing (NLP):** Semantic intent detection using TensorFlow Universal Sentence Encoder for fallback when rule-based matching fails.
- **Rule-Based Responses:** Fast, keyword-based matching for common queries (e.g., "hostel", "fees").
- **Hinglish Support:** Normalization of Hinglish phrases (e.g., "kitna" to "how much").
- **Spell Correction:** Uses Levenshtein distance to correct typos in user input.
- **Conversation Memory:** Remembers last few interactions for context-aware responses.
- **Structured Data:** Provides JSON-structured responses for hostel details (boys/girls).
- **Web Interface:** Static HTML pages served via Express, with JavaScript for chat functionality.
- **CORS Enabled:** Allows cross-origin requests for API access.
- **Gender-Specific Hostel Info:** Differentiates responses for boys' and girls' hostels.
- **Context-Aware Follow-ups:** E.g., after admission query, suggests hostel info.

## Installation and Setup
1. **Prerequisites:** Node.js (v14+), npm installed.
2. **Clone/Download:** Place the project in `d:/chatbot-nlp`.
3. **Install Dependencies:** Run `npm install` in the project root.
4. **Start Server:**
   - Production: `npm start` (runs `node app.js`)
   - Development: `npm run dev` (uses nodemon for auto-restart)
5. **Access:** Open `http://localhost:5000` in browser. Server binds to `0.0.0.0:5000` for external access.

## Usage
- **Web Interface:** Navigate to the served pages. Use the chat interface on `index.html` or other pages to ask questions like "What are the courses?" or "Hostel for girls?".
- **API Direct:** Send POST requests to `/ask` with JSON body `{"question": "your query"}`.
- **Examples:**
  - Input: "admission kaise kare" → Normalized to "admission how to do" → Response: Admission process details.
  - Input: "hostel for boys" → Structured response with boys' hostel data.
- **Supported Queries:** Admissions, fees, courses, hostel (boys/girls), facilities, placements, library, timings, greetings.

---

## API Endpoints
- **POST /ask:** Main chatbot endpoint.
  - Request: `{"question": "string"}`
  - Response: `{ "reply": "string", "structured": {...}, "source": "rule/nlp", ... }`
  - Handles normalization, spell correction, rule matching, NLP fallback.

Static files served from `/public` (e.g., GET /index.html).

## NLP and Logic Explanation
- **Intent Data:** Predefined intents with example phrases (e.g., admission: ["how to apply", "admission process"]).
- **Normalization:** Maps Hinglish to English (e.g., "kya" → "what").
- **Spell Correction:** Corrects words against intent keywords using Levenshtein distance (threshold: 2).
- **Rule-Based Matching:** Keyword checks for fast responses (e.g., if "hostel" in input, return hostel info).
- **NLP Fallback:** Embeds user input and compares to intent centroids using cosine similarity (threshold: 0.60).
- **Memory:** Stores last 6 messages for context.
- **Centroids:** Computed at startup by averaging embeddings of intent examples.

## Challenges and Solutions
- **Challenge:** Handling Hinglish and typos.
  - **Solution:** Normalization map and Levenshtein correction.
- **Challenge:** Semantic matching without full ML training.
  - **Solution:** Universal Sentence Encoder for embeddings and centroids.
- **Challenge:** Context in conversations.
  - **Solution:** Conversation memory array.
- **Challenge:** Structured data for hostels.
  - **Solution:** JSON objects for boys/girls hostels.
